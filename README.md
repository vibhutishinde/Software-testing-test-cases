# Software Testing Project

##  Project Overview
This project demonstrates my manual testing skills on an E-commerce web application. It includes test scenarios, test cases, bug reports, and test execution to simulate a real-world QA workflow.

---

##  What I Tested
- Login functionality (valid & invalid scenarios)
- Form validations (input fields, error handling)
- Navigation flow across pages
- Error messages and validation messages
- Basic user interactions

---

##  Types of Testing Performed
- Functional Testing
- UI Testing
- Regression Testing (after bug fixes)
- Boundary Value Analysis (BVA)
- Equivalence Partitioning (ECP)

---

##  Bug Reporting
- Created detailed bug reports including:
  - Bug ID
  - Clear bug title
  - Steps to reproduce
  - Expected vs Actual result
  - Severity & Priority
  - Environment details

---

##  Tools Used
- Excel (Test case design & bug tracking)
- Postman (API testing)
- GitHub (version control & project hosting)

---

##  Test Artifacts Included
- Test Scenarios
- Test Cases
- Bug Reports
- Test Execution Report (Pass/Fail status)

---

##  Objective
To demonstrate a clear understanding of the Software Testing Life Cycle (STLC), defect reporting process, and real-world QA practices.

---

##  Key Highlights
- Designed 30+ test cases with good coverage
- Identified and documented multiple defects
- Applied testing techniques like BVA & ECP
- Followed structured QA workflow similar to industry practices
